What is agentckpt?

A Python middleware library that adds checkpoint-recovery semantics to LLM agent pipelines. It provides content-addressable state snapshots organized in a git-style DAG, enabling rollback, branching, and deterministic replay of multi-step agent execution. Framework-agnostic — it wraps any async pipeline with composable middleware for checkpointing, retry, idempotent tool execution, and branch-and-merge strategies.

A Python library that gives AI assistants a "save game" feature. When an AI agent is working through a multi-step task, things can go wrong — the AI might make a mistake, an API call might fail, or the whole program might crash. Without agentckpt, you'd have to start over from scratch.

With agentckpt, the AI's progress is saved at each step. If something breaks, it can roll back to the last good save point and try again. It can even explore multiple approaches at the same time and pick the best one.

Imagine you're playing a really hard video game. You get to a tricky part, and you save your game — just in case. If you mess up, you load your save and try again without starting the whole game over.

agentckpt does the same thing, but for AI helpers. When a computer's AI assistant is doing a big job with lots of steps, agentckpt saves its progress along the way. If something goes wrong, it can go back to the last save and keep going.

What problem does it solve?

LLM agent pipelines are stateful, multi-step, and failure-prone. A typical agent loop — observe, think, act, repeat — can involve dozens of LLM calls and tool invocations. Any step can fail. Without checkpointing:

  • A failure at step 47 means re-running steps 1–46 (wasting tokens and time)
  • Non-idempotent tool calls (sending emails, creating records) get duplicated on retry
  • There's no way to inspect or replay the execution history
  • Branching strategies (majority vote, best-of-N) require manual state management

agentckpt solves all of these by treating agent state as a first-class, serializable, version-controlled object.

AI agents work through tasks step by step — browsing the web, calling APIs, writing code, checking results. That could be 10, 20, or 50 steps.

Things break all the time:

  • The internet connection drops mid-task
  • An API goes down temporarily
  • The AI makes a mistake and goes down the wrong path
  • The whole program crashes

Without save points, the AI starts completely over. That wastes time and money. Worse — if the AI already sent an email before the crash, it might send it again on retry. agentckpt prevents all of this.

Think about writing a really long book report. You've been working for an hour, and suddenly your computer shuts off. If you didn't save, you lost everything!

AI helpers have the same problem. They work on big jobs step by step, and if something goes wrong in the middle, they lose all their work. agentckpt makes sure the AI saves its work, can go back if something breaks, and never accidentally does the same thing twice.

Key Features

Checkpoint DAG

Content-addressable snapshots (SHA-256) in a DAG. Branch, merge, and traverse history like git.

Middleware Chain

ASGI-style composable middleware — checkpoint, retry, logging, or write your own.

Idempotent Tools

Classify tools as PURE, SIDE_EFFECT, or NON_DETERMINISTIC. Record/replay for deterministic tests.

Branch & Merge

O(1) copy-on-write forking. Majority vote, best-of-N, three-way merge, beam search.

Pluggable Storage

In-memory, SQLite, filesystem. Implement CheckpointStore for custom backends.

Test Without API Keys

MockLLM, test harness, Hypothesis strategies. 202 tests, 90% coverage.

Save Points

State saved at each step. Roll back, branch off, or inspect what the AI was thinking.

Error Recovery

Failures automatically roll back and retry. Everything is logged.

Smart Tool Handling

Tools don't accidentally run twice. Searches cache; emails send exactly once.

Parallel Exploration

Fork into multiple copies, each trying a different approach. Best answer wins by vote.

Flexible Storage

Store in memory, SQLite, or files on disk. Add your own backend too.

Easy Testing

Fake AI and tools built in. Test without API calls or internet.

Save Your Progress

Like saving in a video game. The AI saves after each step.

Undo Mistakes

Go back to the last save and try again. Like pressing undo.

No Double-Doing

The AI remembers what it already did and won't repeat it by accident.

Try Multiple Ideas

Split into copies, each trying something different. Pick the best answer.

Works Anywhere

Save files can live on your computer, in a database, or as regular files.

Practice Mode

Test everything with a pretend AI. No real AI needed.

How It Works

User Input
    |
    v
[ Pipeline ]
    |
    +--[ LoggingMiddleware ]--+
    |                         |
    +--[ RetryMiddleware ]----+
    |                         |
    +--[ CheckpointMW ]------+
    |                         |
    v                         v
[ Step Function ]      [ Checkpoint DAG ]
    |                    /      |      \
    v                  cp1    cp2     cp3
[ AgentState ]        (main) (main) (branch-a)

The Pipeline drives execution. Each step passes through the middleware chain (onion pattern). CheckpointMiddleware snapshots state before each step and rolls back on failure. The DAG stores the full history as a content-addressed graph. State is a Pydantic model; deps are injected via PipelineContext and not serialized. 

You give the AI a task
        |
        v
   [ Pipeline ] -- the manager that runs everything
        |
        +-- Logs what's happening
        +-- Retries if something fails
        +-- Saves progress automatically
        |
        v
   AI does one step of work
        |
        v
   Progress saved as a checkpoint
        |
        v
   Repeat until done (or roll back if needed)

Think of the Pipeline as a manager overseeing the AI's work. Before the AI does anything, the manager saves the current state. If the step succeeds, move on. If it fails, the manager loads the save and retries. Every step is logged. 

You ask the AI to do something
        |
        v
  The AI's helper saves the game
        |
        v
  The AI does one step of its job
        |
     Did it work?
    /          \
  Yes           No
   |             |
   v             v
 Save again   Load last save,
 Keep going    try again

It's like having a helper standing next to you while you play a game. Before every tricky part, your helper saves for you. If you make it through, your helper saves again. If you fail, your helper loads the save so you can try again.

Getting Started

Install

pip install agentckpt
# or with SQLite backend
pip install agentckpt[sqlite]

Basic Pipeline

import asyncio
from agentckpt import (
    AgentState, Pipeline, PipelineContext,
    CheckpointMiddleware, RetryMiddleware,
    EventLog, Message, Role,
)
from agentckpt.storage.memory import InMemoryCheckpointStore

async def main():
    store = InMemoryCheckpointStore()
    event_log = EventLog()
    ctx = PipelineContext(state=AgentState())

    pipeline = Pipeline(
        ctx,
        middlewares=[
            CheckpointMiddleware(store, event_log=event_log),
            RetryMiddleware(max_retries=2),
        ],
        store=store,
        event_log=event_log,
    )

    async def step(ctx):
        msgs = [*ctx.state.messages,
                Message(role=Role.USER, content="Hello")]
        ctx.state = ctx.state.model_copy(
            update={"messages": msgs}
        )
        return ctx

    await pipeline.execute_step(step)
    cp = pipeline.create_checkpoint()
    print(f"Saved: {cp.id[:12]}...")

asyncio.run(main())

Install it

pip install agentckpt

The basic flow is:

  1. Create a Pipeline to manage the AI's execution
  2. Add middleware for checkpointing, retries, and logging
  3. Define steps — functions that do one unit of work
  4. Run it — saving, error recovery, and logging happen automatically

A programmer types a command to download agentckpt: 

pip install agentckpt

Then they write code that tells the AI: "Here's your job. Do it step by step. I'll save your progress for you." The programmer doesn't have to write the saving code themselves — agentckpt handles it automatically.
