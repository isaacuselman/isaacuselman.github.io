What is agentckpt?

agentckpt is a Python middleware library that adds checkpoint-recovery semantics to LLM agent pipelines. It provides content-addressable state snapshots organized in a git-style DAG, enabling rollback, branching, and deterministic replay of multi-step agent execution. Framework-agnostic — it wraps any async pipeline with composable middleware for checkpointing, retry, idempotent tool execution, and branch-and-merge strategies.

agentckpt is a Python library that gives AI assistants a "save game" feature. When an AI agent is working through a multi-step task (like researching a topic, writing code, or planning a trip), things can go wrong — the AI might make a mistake, an API call might fail, or the whole program might crash. Without agentckpt, you'd have to start over from scratch.

With agentckpt, the AI's progress is saved at each step. If something breaks, it can roll back to the last good save point and try again. It can even explore multiple approaches at the same time and pick the best one.

Imagine you're playing a really hard video game. You get to a tricky part, and you save your game — just in case. If you mess up, you load your save and try again without starting the whole game over.

agentckpt does the same thing, but for AI helpers. When a computer's AI assistant is doing a big job with lots of steps, agentckpt saves its progress along the way. If something goes wrong, it can go back to the last save and keep going. No starting over!

What problem does it solve?

LLM agent pipelines are stateful, multi-step, and failure-prone. A typical agent loop — observe, think, act, repeat — can involve dozens of LLM calls and tool invocations. Any step can fail: rate limits, network timeouts, malformed outputs, tool errors. Without checkpointing:

  • A failure at step 47 means re-running steps 1–46 (wasting tokens and time)
  • Non-idempotent tool calls (sending emails, creating records) get duplicated on retry
  • There's no way to inspect or replay the execution history
  • Branching strategies (majority vote, best-of-N) require manual state management

agentckpt solves all of these by treating agent state as a first-class, serializable, version-controlled object.

AI agents don't just answer one question — they work through tasks step by step. An AI might browse the web, call an API, write some code, then check if it worked. That could be 10, 20, or even 50 steps.

The problem? Things break all the time:

  • The internet connection drops mid-task
  • An API the AI is using goes down temporarily
  • The AI makes a mistake and goes down the wrong path
  • The whole program crashes

Without save points, the AI has to start completely over. That wastes time and money (AI API calls cost real money). Worse — if the AI already sent an email or created a file before the crash, it might send it again when it retries. agentckpt prevents all of this.

Think about writing a really long book report. You've been working for an hour, and suddenly your computer shuts off. If you didn't save, you lost everything!

AI helpers have the same problem. They work on big jobs step by step, and if something goes wrong in the middle, they lose all their work. Even worse — imagine the AI was supposed to send one birthday card to your friend, but after the crash it sends it again. Now your friend got two cards! Awkward.

agentckpt makes sure the AI saves its work, can go back if something breaks, and never accidentally does the same thing twice.

Key Features

Checkpoint DAG

Content-addressable state snapshots (SHA-256) in a directed acyclic graph. Branch, merge, and traverse history like git. Checkpoints are immutable Pydantic models — identical states share the same ID automatically.

Middleware Chain

ASGI-style onion middleware — compose CheckpointMiddleware, RetryMiddleware, and LoggingMiddleware (or write your own). Each step passes through the full chain.

Idempotent Tools

Classify tools as PURE (cache forever), SIDE_EFFECT (execute-once with idempotency key), or NON_DETERMINISTIC (TTL-based cache). Includes a record/replay system for deterministic test playback from JSONL.

Branch & Merge

O(1) copy-on-write state forking. Run parallel branches with majority vote, best-of-N selection, three-way dict merge, or MAKER-style first-to-ahead-by-K voting. Built-in beam search over agent states.

Pluggable Storage

In-memory (testing), SQLite (local/production), filesystem (content-addressed, human-inspectable). Implement the CheckpointStore protocol for custom backends.

Test Without API Keys

Ships with MockLLM, test harness, and Hypothesis strategies. 202 tests, 90% coverage, fully offline.

Save Points (Checkpoints)

The AI's state is saved at each step, forming a tree of save points. You can roll back to any previous point, branch off to try different approaches, or inspect exactly what the AI was thinking at any moment.

Automatic Error Recovery

Middleware automatically handles failures. If a step fails, the state rolls back and can retry with exponential backoff. Everything is logged so you can see what happened.

Smart Tool Handling

Tools (like web search or sending emails) are wrapped so they don't accidentally run twice. A search can be cached; an email send is guaranteed to only happen once even if the system retries.

Parallel Exploration

The AI can fork into multiple copies, each trying a different approach. Then the best answer wins — by voting, scoring, or combining results. Like asking three friends the same question and going with the majority answer.

Flexible Storage

Save points can be stored in memory (for testing), in a SQLite database (for real use), or as files on disk. You can add your own storage backend too.

Easy Testing

Comes with fake AI and fake tools built in, so you can test everything without paying for API calls or needing internet access.

Save Your Progress

Just like saving in a video game. The AI saves after each step so it never loses its work.

Undo Mistakes

If the AI messes up, it can go back to the last save and try again. Like pressing the undo button.

No Double-Doing

If the AI already sent a message or looked something up, it remembers and won't do it again by accident.

Try Multiple Things at Once

The AI can split into copies of itself, each trying a different idea. Then it picks the best answer. Like asking three friends and going with what most of them say.

Works Anywhere

The save files can be stored in different places — on your computer, in a database, or as regular files you can look at.

Practice Mode

Comes with a pretend AI so you can test everything without needing the real thing. Like a practice mode in a game.

How It Works

User Input
    |
    v
[ Pipeline ]
    |
    +--[ LoggingMiddleware ]--+
    |                         |
    +--[ RetryMiddleware ]----+
    |                         |
    +--[ CheckpointMW ]------+
    |                         |
    v                         v
[ Step Function ]      [ Checkpoint DAG ]
    |                    /      |      \
    v                  cp1    cp2     cp3
[ AgentState ]        (main) (main) (branch-a)

The Pipeline drives execution. Each step passes through the middleware chain (onion pattern — outer middleware wraps inner). CheckpointMiddleware snapshots state before each step and rolls back on failure. The CheckpointDAG stores the full history as a content-addressed graph. State is a Pydantic model; deps (LLM, tools, store) are injected via PipelineContext and not serialized. 

You give the AI a task
        |
        v
   [ Pipeline ] ← the manager that runs everything
        |
        +-- Logs what's happening
        +-- Retries if something fails
        +-- Saves progress automatically
        |
        v
   AI does one step of work
        |
        v
   Progress is saved as a checkpoint
        |
        v
   Repeat until done (or roll back if needed)

Think of the Pipeline as a manager overseeing the AI's work. Before the AI does anything, the manager saves the current state. The AI does its work. If it succeeds, great — move on. If it fails, the manager loads the save and either retries or lets you decide what to do. Every step is logged, so you have a complete record of what happened and when. 

You ask the AI to do something
        |
        v
  The AI's helper saves the game
        |
        v
  The AI does one step of its job
        |
     Did it work?
    /          \
  Yes           No
   |             |
   v             v
 Save again   Load last save,
 Keep going    try again

It's like having a helper standing next to you while you play a game. Before every tricky part, your helper saves the game for you. If you make it through, awesome — your helper saves again and you keep going. If you fail, your helper loads the save so you can try again. Your helper also writes down everything that happens in a notebook, so you can look back later and see exactly what you did.

Getting Started

Install

# or with SQLite backend
pip install agentckpt[sqlite]

Basic Pipeline

import asyncio
from agentckpt import (
    AgentState, Pipeline, PipelineContext,
    CheckpointMiddleware, RetryMiddleware,
    EventLog, Message, Role,
)
from agentckpt.storage.memory import InMemoryCheckpointStore

async def main():
    store = InMemoryCheckpointStore()
    event_log = EventLog()
    ctx = PipelineContext(state=AgentState())

    pipeline = Pipeline(
        ctx,
        middlewares=[
            CheckpointMiddleware(store, event_log=event_log),
            RetryMiddleware(max_retries=2),
        ],
        store=store,
        event_log=event_log,
    )

    async def step(ctx):
        msgs = [*ctx.state.messages,
                Message(role=Role.USER, content="Hello")]
        ctx.state = ctx.state.model_copy(
            update={"messages": msgs}
        )
        return ctx

    await pipeline.execute_step(step)
    cp = pipeline.create_checkpoint()
    print(f"Saved: {cp.id[:12]}...")

asyncio.run(main())

Use it

The basic flow is:

  1. Create a Pipeline — this manages the AI's execution
  2. Add middleware — plugins that add checkpointing, retries, and logging
  3. Define steps — functions that do one unit of work
  4. Run it — the pipeline handles saving, error recovery, and logging automatically

The code is in Python and uses async/await (a way to write code that can wait for slow operations without freezing). If you've taken AP Computer Science or done any Python, the patterns will feel familiar.

To use agentckpt, a programmer types a special command to download it to their computer: 

pip install agentckpt

Then they write code that tells the AI: "Here's your job. Do it step by step. I'll save your progress for you." The cool part is that the programmer doesn't have to write all the saving and loading code themselves — agentckpt handles it automatically!

It's like getting a free helper that does all the boring save-and-load work so the programmer can focus on the fun stuff.
